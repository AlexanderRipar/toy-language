Only files th extension `.evl` are treated as test cases.

The expected outcome of each test is indicated by a line comment that must
appear before any code or other comments in the file. The format of this
header is `<expected-result>[:<line-number>[:<column-number]]`, where
`<expected-result>` is either "success" if the test case is expected to
succeed, or the name of a `CompileError` enumerant which is expected to be
among the errors generated by the test.

If `<line-number>` or `column-number` are specified, errors are only considered
as matching when their line and column numbers match the ones specified.
Otherwise, any error matching `<expected-result>` is treated as a match,
regardless of its origin.

Note that the file in which the error is expected to occur cannot currently be
specified and is always ignored.
